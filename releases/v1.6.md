# v1.6 — Strategy Configurator

Status: ⚪ Planned

**Motivation:** v1.4 exposes 5 tunable strategy parameters (forgiveness, retaliation_delay, noise_tolerance, initial_moves, cooperate_bias), but the frontend has no UI for them. Players must manually construct instruction data to set non-default params. The Strategy Configurator is an interactive frontend component that lets players explore, understand, and configure strategy parameters visually — then generates the exact instruction bytes or CLI command to enter the tournament.

**Depends on:** v1.4 (Strategy Parameters) deployed.

---

## Goals

1. Make strategy parameters **discoverable** — players learn what params exist and which ones matter for their chosen strategy
2. Make parameter tuning **intuitive** — sliders, visual feedback, and plain-English descriptions
3. Generate **ready-to-use output** — instruction bytes, CLI command, or copyable code snippet
4. Show **behavioral previews** — simulate how the configured strategy plays against reference opponents
5. Work entirely **client-side** — no backend changes, pure frontend feature

## 2. Strategy Selector Component (`web/src/components/StrategySelector.tsx`)

Row of strategy buttons using existing `StrategyBadge` styling. Selected strategy is highlighted. Clicking a strategy:

- Updates the Parameter Panel to show only **relevant** parameters
- Resets params to defaults for that strategy
- Triggers preview recalculation

### Relevance Map

| Strategy | Relevant Params |
|----------|----------------|
| TitForTat | forgiveness, retaliation_delay, initial_moves |
| AlwaysDefect | initial_moves |
| AlwaysCooperate | initial_moves |
| GrimTrigger | noise_tolerance, initial_moves |
| Pavlov | initial_moves |
| SuspiciousTitForTat | forgiveness, retaliation_delay, initial_moves |
| Random | cooperate_bias, initial_moves |
| TitForTwoTats | forgiveness, initial_moves |
| Gradual | initial_moves |

Non-relevant params are hidden (not grayed out — hidden). Keeps the UI clean. `initial_moves` is always shown since it applies universally.

## 3. Parameter Panel Component (`web/src/components/ParamPanel.tsx`)

Each parameter rendered as a labeled control with:

- **Name** (human-readable)
- **Description** (one-liner explaining the effect)
- **Control** (slider or specialized input)
- **Current value** (numeric display beside slider)

### Parameter Controls

| Parameter | Control | Range | Step | Display |
|-----------|---------|-------|------|---------|
| `forgiveness` | Slider | 0–100 | 1 | `{value}%` — "Chance to forgive a defection" |
| `retaliation_delay` | Slider | 0–10 | 1 | `{value} rounds` — "Rounds to wait before retaliating" |
| `noise_tolerance` | Slider | 0–5 | 1 | `{value} defections` — "Defections to ignore before triggering" |
| `cooperate_bias` | Slider | 0–100 | 1 | `{value}%` — "Cooperation probability per round" |
| `initial_moves` | 8-toggle grid | bit 0–7 | toggle | Visual: 8 round boxes, green=C / red=D — "Override first 8 rounds" |

### `initial_moves` Toggle Grid

8 small boxes in a row, each representing rounds 0–7. Click to toggle C↔D. Default: all Cooperate (0x00). Displays the bitmask value below as hex.

```
Round:  0   1   2   3   4   5   6   7
       [C] [C] [D] [C] [C] [C] [C] [C]  = 0x04
```

Green box = Cooperate (bit=0), Red box = Defect (bit=1).

### Preset Buttons

Below the sliders, show named presets for the selected strategy:

| Strategy | Presets |
|----------|---------|
| TitForTat | "Classic" (defaults), "Generous TFT" (forgiveness=30), "Forgiving TFT" (forgiveness=10) |
| GrimTrigger | "Classic" (defaults), "Tolerant Grim" (noise_tolerance=2), "Patient Grim" (noise_tolerance=4) |
| Random | "Fair Coin" (bias=50), "Mostly Cooperate" (bias=80), "Mostly Defect" (bias=20) |
| SuspiciousTfT | "Classic" (defaults), "Generous Suspicious" (forgiveness=20) |

Presets are buttons that set slider values. Players can further adjust after selecting a preset.

## 4. Preview Panel Component (`web/src/components/StrategyPreview.tsx`)

Runs a **client-side simulation** of the configured strategy against all 9 base strategies (default params). This is the core "is my config any good?" feedback loop.

### Implementation

Port `match_logic::run_match` to TypeScript (or compile match-logic crate to WASM — see item 8). Runs 9 simulations (configured strategy vs each base strategy), 10 rounds each, deterministic seed.

### Display

**Summary table:**

```
Opponent              Score   Outcome
─────────────────────────────────────
vs Tit for Tat         30    ██████████ (mutual cooperation)
vs Always Defect       13    ████░░░░░░ (exploited early)
vs Always Cooperate    50    ██████████████████ (dominant)
vs Grim Trigger        30    ██████████ (stable)
...
Total: 287 / 450
```

Score bar is colored relative to max possible (50 per match = 5 × 10 rounds).

**Move-by-move detail (expandable):**

Click any row to expand and see the round-by-round moves:

```
vs Always Defect (10 rounds):
  R1: C↔D (0,5)  R2: D↔D (1,1)  R3: D↔D (1,1)  ...
  Your score: 13  Opponent: 14
```

Color-coded: C=green, D=red. Shows payoffs per round.

### Random Strategy Handling

For `Random` or any strategy with stochastic params (forgiveness > 0), run N=100 simulations and show **average ± std dev** instead of a single deterministic result.

## 5. Output Panel Component (`web/src/components/ConfigOutput.tsx`)

Three output formats, switchable via tabs:

### CLI Command

```bash
arena enter <WALLET_PATH> tit-for-tat --forgiveness 30 --initial-moves 0x04
```

Only includes non-default flags. Copyable.

### Instruction Bytes

```
Data: [strategy_index, forgiveness, retaliation_delay, noise_tolerance, initial_moves, cooperate_bias]
       [0, 30, 0, 0, 4, 0]
Hex:  001e00000400
```

### TypeScript Snippet

```typescript
const data = Buffer.from([
  0,   // strategy: TitForTat
  30,  // forgiveness
  0,   // retaliation_delay
  0,   // noise_tolerance
  4,   // initial_moves (0b00000100 = defect round 2)
  0,   // cooperate_bias
]);
```

Each format has a copy button. The snippet includes comments mapping each byte to its parameter.

## 6. Strategy Info Tooltips (`web/src/components/StrategyInfo.tsx`)

Expandable info section per strategy in the selector. Shows:

- **Description:** 1–2 sentences (reuse from guide page)
- **Strengths:** What it's good against
- **Weaknesses:** What exploits it
- **Parameter effects:** How params modify the base behavior

Example for Tit for Tat:

> **Tit for Tat** — Start cooperating, then copy opponent's last move.
>
> ✅ Strong against: cooperators, other TFT variants
> ❌ Weak against: initial defectors (recovers slowly without forgiveness)
>
> **forgiveness** → % chance to cooperate instead of retaliating. Higher = more tolerant of noise.
> **retaliation_delay** → Rounds to wait before copying a defection. Creates "patient" TFT.
> **initial_moves** → Override opening rounds. Use to test opponent with early defections.

## 8. Match-Logic WASM Build (Optional)

Instead of porting match logic to TypeScript, compile the `match-logic` crate to WASM for use in the browser preview. This guarantees the preview matches on-chain behavior exactly.

### Setup

```toml
# crates/match-logic/Cargo.toml
[lib]
crate-type = ["cdylib", "rlib"]

[dependencies]
wasm-bindgen = "0.2"
```

### Exported Functions

```rust
#[wasm_bindgen]
pub fn simulate_match(
    strategy_a: u8,
    params_a: &[u8],  // 5 bytes
    strategy_b: u8,
    params_b: &[u8],  // 5 bytes
    rounds: u32,
    seed: u64,
) -> Vec<u8>  // encoded results
```

### Justfile Recipe

```just
build-wasm:
    cd crates/match-logic && wasm-pack build --target web --out-dir ../../web/src/lib/wasm
```

**If WASM adds too much complexity**, fall back to a TypeScript port. The preview is informational — slight behavioral differences from the on-chain logic are acceptable for v1.6. But WASM is preferred if feasible.

## Risk Assessment

- **Zero on-chain risk:** Entirely frontend. No contract changes.
- **Simulation accuracy:** If using TS port, behavior may drift from Rust match-logic over time. WASM eliminates this but adds build complexity. Recommend WASM with TS fallback.
- **Bundle size:** WASM module adds ~50–100KB gzipped. Acceptable. Load lazily on `/configure` route only.
- **Scope creep:** The preview panel (item 7) is the most complex piece. If timeline is tight, ship without move-by-move detail (summary table only) and add detail view in a patch.
- **No new dependencies expected** — sliders use native HTML range inputs styled with Tailwind. No UI library needed.
