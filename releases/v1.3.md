# v1.3 â€” Persist Strategy in Tournament Struct

Status: đźź˘ Implemented

**Motivation:** When a player claims their payout or their entry expires, the Entry PDA is closed and all per-player metadata is lost â€” including their strategy. The frontend scoreboard can only show "Unknown" for claimed winners. This release stores strategy in the Tournament account so it survives entry closure.

---

## 1. Add `strategies` Vec to Tournament Struct (Contract)

**Change:** Add `pub strategies: Vec<u8>` to the `Tournament` struct, alongside `players` and `scores`.

```rust
pub struct Tournament {
    // ... existing fields ...
    pub players: Vec<Pubkey>,
    pub scores: Vec<u32>,
    pub strategies: Vec<u8>,   // NEW â€” index matches players/scores
    pub bump: u8,
}
```

**Size impact:** +1 byte per player + 4 bytes vec length prefix.
- At 100 players: +104 bytes (negligible)
- At 5000 players: +5,004 bytes (~5KB, well within limits)
- `BYTES_PER_PLAYER` changes from 36 â†’ 37

**Fields to update:**
- `BYTES_PER_PLAYER`: 36 â†’ 37
- `Tournament::BASE_SPACE`: add 4 bytes for the new vec length
- `Tournament::space()`: uses `BYTES_PER_PLAYER`, auto-adjusts
- `Tournament::space_for_next_player()`: same

## 2. Update `enter_tournament` (Contract)

Push the strategy value into the new vec when a player enters:

```rust
tournament.strategies.push(entry.strategy as u8);
```

This mirrors the existing `tournament.players.push(...)` and `tournament.scores.push(0)`.

---

## 3. Update `claim_refund` (Contract)

When a player is refunded, their strategy slot should be zeroed (consistent with `players[index] = Pubkey::default()` and `scores[index]` staying 0):

```rust
tournament.strategies[entry.index as usize] = u8::MAX; // 255 = refunded/invalid
```

Using 255 since all valid strategies are 0-8.

---

## 4. Update `realloc` in `EnterTournament` Accounts (Contract)

The realloc calculation in the `EnterTournament` accounts struct must account for the extra byte per player:

```rust
realloc = Tournament::BASE_SPACE + ((tournament.players.len() + 1) * BYTES_PER_PLAYER),
```

This already uses `BYTES_PER_PLAYER`, so updating the constant is sufficient.

---

## 5. Update `close_registration` Odd-Player Refund (Contract)

When an odd player is refunded at registration close, also mark their strategy slot:

```rust
tournament.strategies[last_index] = u8::MAX;
```

Contract struct change requires redeployment. Since this adds a new vec to Tournament:
- **Existing Tournament PDAs are incompatible** â€” Borsh deserialization will fail on old accounts

- **Recommendation:** Option A for devnet. For mainnet, ensure no active tournaments before upgrade.
